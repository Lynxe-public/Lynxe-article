# AIAgent系列-2 ReAct Agent 与 传统的写流程 对比，有哪些本质不同？

说到 AI Agent，ReAct 算是个绕不开的话题。但很多人可能还没搞清楚，它和咱们平时写代码的传统流程到底有啥不一样。今天就来聊聊这个。

## 传统流程：按部就班

传统的编程流程其实挺直白的。你拿到需求，脑子里过一遍逻辑，然后就开始写代码。整个过程基本是线性的：分析问题、设计算法、写代码、测试、调试。每一步都是你提前想好的，代码执行路径也是固定的。

比如你要写个爬虫，流程大概是：先定义要爬的网站，然后写请求逻辑，解析 HTML，提取数据，最后存到数据库。每一步都是你预先设计好的，程序跑起来就是按这个顺序走，不会有啥意外。

## ReAct Agent：边想边做

ReAct Agent 的核心思想是 "Reasoning and Acting"，也就是边推理边行动。它不是一次性把整个流程都规划好，而是通过观察-思考-行动的循环，根据实际情况动态调整策略。

## 本质区别在哪？

| 对比维度 | 传统流程 | ReAct Agent |
|---------|---------|-------------|
| **决策时机** | 写代码时就定好了所有决策 | 运行时根据实际情况动态决策 |
| **执行方式** | 按预设的固定路径执行 | 观察-思考-行动循环，路径可变 |
| **可预测性** | 输入相同，输出相同 | 每次执行路径可能不同 |
| **适用场景** | 逻辑清晰、需求明确的任务 | 复杂、不确定、需要灵活应对的任务 |
| **比喻** | 按剧本演戏，每一步都排练好 | 即兴表演，根据现场情况调整 |
| **优势** | 稳定可靠，容易调试 | 灵活适应，能处理意外情况 |
| **劣势** | 遇到新情况需要改代码 | 行为可能不可预测，有时会"犯傻" |
